## 🔎 0. 우리 회사가 **EPIA 대상**인지 한눈에 보는 자가 체크리스트

> 아래 항목 중 **단 하나라도 `✔` 체크**된다면,  
> 우리 회사/서비스는 **EPIA 기반 개인정보 영향평가를 검토할 만한 수준의 개인정보 리스크**를 가진 것으로 볼 수 있습니다.

### ✅ 자가 체크리스트
- [ ] **회사의 개인정보 보호 수준을 스스로 점검하고 싶다.**
- [ ] 우리 서비스/업무에서 **민감정보**(건강, 장애, 범죄경력, 금융정보 등)나 **주민등록번호 등 고유식별정보**를 정기적으로 처리하고 있다.
- [ ] 신규 서비스 출시, 기능 추가, 레거시 개편 등으로 **개인정보 처리 프로세스가 크게 바뀌었거나 바뀔 예정**이다.
      (예: 시스템 통합/분리, 신규 모듈 추가, 조직 개편에 따른 업무 재설계 등)
- [ ] 사내 시스템을 **클라우드 / 외부 SaaS / 외부 API**와 연계하여 개인정보가 회사 외부로 이동·저장·처리되는 구조가 있다.
- [ ] **AI·빅데이터 분석, 자동 의사결정**(신용평가, 이상거래 탐지, 맞춤 추천 등)에 데이터를 활용하거나 도입을 검토 중이다.
- [ ] 최근 또는 과거에 **개인정보 유출·오남용·오설정 등의 사고/사건**이 있었거나, 비슷한 위험 징후가 반복적으로 발생하고 있다.
- [ ] 회사 내부에서 “**어디서 어떤 개인정보가 어떻게 흐르는지** (수집 → 보유 → 이용 → 제공 → 파기)”를 한 번에 파악하기 어렵다는 인식이 있다.
- [ ] 신규 사업/서비스를 기획할 때마다 “**법적으로 문제 없는지, 데이터 리스크가 어느 정도인지**”를 매번 **사후에 수작업으로만 검토**하고 있다.
- [ ] 경영진 또는 CISO/CPO가 **개인정보 이슈가 사업 리스크(평판, 계약, 규제, 투자 등)로 번질 수 있다고 우려**하고 있다.

> [!TIP]
> 위 항목 중 **2개 이상에 해당**된다면,  
> EPIA를 도입해  
> **개인정보 흐름분석, 이행관리, 침해요인별 보호조치, 개선과제**를  
> 정형화된 **개인정보 영향평가(PIA) 프로세스**로 관리하는 것을 권장합니다.
